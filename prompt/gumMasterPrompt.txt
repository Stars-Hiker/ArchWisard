LAYER 1 — THE GUM TTY ESCAPE HATCH  (critical for piped stdout)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a script runs  exec > >(tee logfile) 2>&1  stdout is no longer a
tty.  Gum detects this and either renders nothing or strips ANSI.

T1  Display calls (gum style, gum log, gum spin display):
    Write to /dev/tty explicitly via a helper:
      _tty() { [[ ! -t 1 ]] && [[ -w /dev/tty ]] && echo /dev/tty || echo 1; }
    Then:  gum style "…" > "$(_tty)"  2>/dev/null || printf '%s\n' "…"

T2  Interactive calls (gum choose, gum input, gum confirm, gum filter):
    Redirect both stdin and stderr to /dev/tty so gum gets a real
    terminal for keyboard input AND rendering:
      RESULT=$(gum choose … </dev/tty 2>/dev/tty)
    The selection is still captured by $() on stdout.

T3  NEVER embed $(gum style …) inside --title, --header, or another
    gum call.  Use the inline ANSI helper instead:
      _c() { printf '\033[38;5;%sm%s\033[0m' "$1" "$2"; }
      # Usage: gum style "$(_c 141 "highlight") plain text"

T4  NEVER pass --selected "" to gum choose — it exits non-zero.
    Guard with a match check before using --selected.

gum style
  - Use for all banners, bordered panels, and status lines.
  - Compose multi-column layouts with gum join --horizontal:
      LEFT=$(gum style --border rounded --width 34 "$col1_content")
      RIGHT=$(gum style --border rounded --width 34 "$col2_content")
      gum join --horizontal "$LEFT" "$RIGHT" > "$(_tty)"
  - Stack panels vertically with gum join --vertical.
  - --align center/right for headers; --align left (default) for body.
  - Prefer --padding "0 2" over manual spaces.

gum log  (REPLACES hand-rolled ok/warn/error/info functions)
  - Use for ALL runtime messages.  Levels: debug info warn error fatal
      gum log --level info  "Packages installed"
      gum log --level warn  "EFI < 260 MB"
      gum log --level error "Disk not found: $disk" 2>/dev/null
      gum log --structured --level debug "Probing" disk "$d" size "$s"
  - Add  --time timeonly  for long-running scripts.
  - Direct to /dev/tty via: gum log … > "$(_tty)" 2>/dev/null

gum spin
  - Wrap every long-running command:
      gum spin --spinner dot --title " Installing…" \
        -- bash -c "pacstrap /mnt $pkgs 2>&1 | tee -a $LOG" \
        </dev/tty 2>/dev/tty
  - Spinner styles: dot line minidot jump pulse points globe moon
  - Run the inner command in a subshell when it needs variables:
      gum spin … -- bash -c "eval \"$cmd\""

gum choose / gum filter
  - choose: bounded list, known items, up to ~20 options.
  - filter: fuzzy-searchable, use for >20 items or filenames.
  - Always </dev/tty 2>/dev/tty and capture to a variable.
  - Provide a sensible --height (8–16); never let it default to full screen.
  - Use --no-limit for multi-select; pipe to mapfile for arrays:
      mapfile -t SELECTED < <(gum choose --no-limit … </dev/tty 2>/dev/tty)

gum confirm
  - Every destructive action. Returns exit code 0 (yes) / 1 (no).
  - Use with explicit if/then/fi (never && under set -e):
      if gum confirm "Erase $DISK?" </dev/tty 2>/dev/tty; then …; fi

gum input / gum write
  - input: single-line values.  write: multi-line (Ctrl+D to submit).
  - Always provide --placeholder and --header for context.
  - Capture: VAR=$(gum input --header "Hostname" … </dev/tty 2>/dev/tty)

gum table
  - Render structured data (disk lists, package summaries) from CSV:
      printf 'DEVICE,SIZE,TYPE\n%s,%s,%s\n' … |
        gum table --border rounded --widths 16,8,6 > "$(_tty)"

gum format
  - Render Markdown inline:  printf '# Title\n- item\n' | gum format
  - Render Lip Gloss templates without subshells:
      gum format -t template \
        '{{ Bold "Step" }} {{ Color "141" "0" " 3/8 " }} Storage stack'
  - Syntax-highlight code blocks: cat script.sh | gum format -t code

gum pager
  - Use for any output >30 lines (help text, logs, summaries):
      gum pager < /tmp/install.log

LAYER 4 — BASH 5.x PERFORMANCE & BREVITY RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

B1  Subshell elimination (every $() is a fork — eliminate them in loops):
      $(<file)           not  $(cat file)
      cmd <<< "$var"     not  echo "$var" | cmd
      ${#arr[@]}         not  $(echo "${arr[@]}" | wc -w)
      (( n > 5 ))        not  [[ $(cmd) -gt 5 ]]

B2  String ops via parameter expansion, not sed/awk for simple cases:
      ${var#prefix}   ${var%suffix}   ${var//old/new}
      ${var^^}  ${var,,}  ${var@Q}   (Bash 5: uppercase/lower/quote)

B3  Arrays — always use mapfile/readarray for multi-line output:
      mapfile -t lines < <(some_command)   # no subshell, no IFS games

B4  Nameref (declare -n) for passing arrays by reference — zero subshells:
      build_list() { declare -n _out=$1; _out=("a" "b" "c"); }
      build_list MY_ARRAY

B5  Arithmetic — use (( )) and $(( )), never expr or bc for integers:
      (( total = free + disposable ))
      if (( total < NEEDED )); then …; fi

B6  Batch external calls — one blkid/lsblk pass per disk, not one per field:
      read -r UUID FSTYPE < <(blkid -o value -s UUID -s TYPE "$part")

B7  Printf over echo for any string containing variables or escape chars.

B8  Local variables in every function: declare all with  local  up front.
    Use  local -r  for constants,  local -i  for integers (implicit (())):
      local -i total=0
      local -r MIN=7

B9  Function contracts — one comment per function, no more:
      # inputs: $1=disk $2=size_gb | sets: ROOT_PART EFI_PART | ⚠ writes disk

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every interactive wrapper MUST have a NO_GUM=true plain-read path.
Pattern:

  _pick() {
    # inputs: $1=prompt, rest=choices | outputs: selected item on stdout
    if [[ "${NO_GUM:-false}" == true ]]; then
      local i=1 item
      for item in "${@:2}"; do printf '  %d) %s\n' "$i" "$item" >&2; (( i++ )); done
      local n; read -rp "Choice [1-$(( i-1 ))]: " n </dev/tty
      echo "${@: $(( n+1 )):1}"; return
    fi
    gum choose --header "$1" --height 10 "${@:2}" </dev/tty 2>/dev/tty
  }

Auto-detect and degrade gracefully:
  command -v gum &>/dev/null || NO_GUM=true
